Configuration

How to configure anchart3d charts to your liking

Once you're done with the basic setup, as described here, you can start configuring using the fluent api based methods as described below.

You first need to create a configuration object that holds a set of configurations. var config = {param1:value1, param2:value2}

The configuration object has to be passed to the configuration() method of the fluent api, as seen in the section usage.

Now let's have a look at the possible parameters and how they affect the chart in the canvas.
Hint: The following options are all treated as booleans in the library (except the option "bgcolor"). Set them to true or false for either enabling or disabling the specific feature.

antialias
Smoother visuals (e.g. smoothing out the edges on polygons) but can cause performance issues on old computers.
chartAnimation
Set this parameter to true in order to enable chart specific animations. (scatter chart is excluded!)
details
When a segement/entity gets clicked, a detail pane is displayed.
fov
Set the field of view for the camera. The default fov is 45.
legend
Displays a legend.
showOnScreenControls
Usually the chart can be interacted with using the mouse. Setting this option to true will display an element, that allows moving the chart with a D-Pad.
startAnimation
Wheter an entry animation should be used when the page is loaded or not. Setting this option to false means, that the chart will be immediately displayed without an entry animation.
tooltip
When the cursor hovers on a segment/entity, a tooltip is displayed.
transparency
Enable or disable the transparency of the canvas by setting this option to true or false
bgcolor
This option changes the background color of the canvas depending on the value set to the property itself.
It can take different color inputs:
  • Hexadecimal (recommended): Set the color with a hexadecimal number (e.g. "0xff0000")
  • RGB string: Set the color with a RGB string (e.g. "rgb(255, 0, 0)" or "rgb(100%, 0%, 0%)")
  • X11 color name: Set the color with the X11 color name. All 140 colors are supported. (e.g. "red" or "skyblue")
  • HSL string: Set the color with a HSL string. (e.g. "hsl(0, 100%, 50%)")
